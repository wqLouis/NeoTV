# Internationalization (i18n)

## Overview

NeoTV uses [inlang](https://inlang.com) for internationalization. The project supports:

- English (`en`) - base locale
- Chinese Simplified (`zh-cn`)

## Project Structure

```
project.inlang/
├── settings.json       # inlang configuration
├── cache/             # Plugin cache (gitignored)
├── .gitignore
└── README.md

messages/
├── en.json            # English translations
└── zh-cn.json         # Chinese translations
```

## Configuration

**File:** `project.inlang/settings.json`

```json
{
	"$schema": "https://inlang.com/schema/project-settings",
	"modules": [
		"https://cdn.jsdelivr.net/npm/@inlang/plugin-message-format@4/dist/index.js",
		"https://cdn.jsdelivr.net/npm/@inlang/plugin-m-function-matcher@2/dist/index.js"
	],
	"plugin.inlang.messageFormat": {
		"pathPattern": "./messages/{locale}.json"
	},
	"baseLocale": "en",
	"locales": ["en", "zh-cn"]
}
```

## Translation Files

### English (`messages/en.json`)

```json
{
	"$schema": "https://inlang.com/schema/inlang-message-format",
	"hello_world": "Hello, {name} from en!"
}
```

### Chinese (`messages/zh-cn.json`)

```json
{
	"$schema": "https://inlang.com/schema/inlang-message-format",
	"hello_world": "你好，{name}！"
}
```

## Adding Translations

### 1. Add Key to Message Files

Add the same key to both locale files:

```json
// messages/en.json
{
	"app_name": "NeoTV",
	"search_placeholder": "Search videos...",
	"settings_title": "Settings"
}
```

```json
// messages/zh-cn.json
{
	"app_name": "NeoTV",
	"search_placeholder": "搜索视频...",
	"settings_title": "设置"
}
```

### 2. Use in Components

```svelte
<script>
	import { message } from '$lib/paraglide/lang';
</script>

<h1>{$message('app_name')}</h1>
<input placeholder={$message('search_placeholder')} />
```

## Message Format Syntax

The project uses the [ICU MessageFormat](https://messageformat.github.io/messageformat/) syntax:

### Simple Interpolation

```json
"greeting": "Hello, {name}!"
```

### Pluralization

```json
"items_count": "{count, plural, =0 {No items} =1 {1 item} other {# items}}"
```

### Select

```json
"gender_message": "{gender, select, male {He} female {She} other {They}} is online"
```

## Current Status

**Note:** The i18n system is currently set up but not actively used throughout the application. The translation files contain only placeholder messages. To fully internationalize the app:

1. Extract all hardcoded strings from components
2. Add them to the message files
3. Replace hardcoded strings with `{$message('key')}` calls

## Development Commands

```bash
# Install inlang SDK (if needed)
npm install @inlang/sdk

# Update message cache
npx inlang langtool

# Check for missing translations
npx inlang lint
```
